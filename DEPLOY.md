# Инструкция по деплою на Railway

## Шаг 1: Подготовка проекта

1. Убедитесь, что все файлы сохранены
2. Убедитесь, что `requirements.txt` содержит все зависимости

## Шаг 2: Создание проекта на Railway

1. Перейдите на https://railway.com/new
2. Войдите через GitHub (или другой способ)
3. Нажмите "New Project"
4. Выберите "Deploy from GitHub repo" (если проект в GitHub) или "Empty Project"

## Шаг 3: Настройка переменных окружения

В настройках проекта Railway добавьте следующие переменные окружения:

### Обязательные:
- `DEEPSEEK_API_KEY` - ваш API ключ DeepSeek
- `TELEGRAM_BOT_TOKEN` - токен вашего Telegram бота
- `CHANNEL_ID` - ID канала для публикации постов (например: -1003546844550)

### Опциональные (для парсинга новостей):
- `TELEGRAM_API_ID` - ваш Telegram API ID (число)
- `TELEGRAM_API_HASH` - ваш Telegram API Hash (строка)

### Остальные (можно оставить значения по умолчанию):
- `NEWS_CHANNEL=markettwits`
- `NEWS_COUNT=3`
- `COINGECKO_API_URL=https://api.coingecko.com/api/v3/simple/price`
- `TON_COIN_ID=the-open-network`

## Шаг 4: Деплой

1. Если деплоите из GitHub:
   - Подключите репозиторий
   - Railway автоматически определит Python проект
   - Нажмите "Deploy"

2. Если деплоите через Railway CLI:
   ```bash
   railway login
   railway init
   railway up
   ```

## Шаг 5: Проверка работы

1. После деплоя проверьте логи в Railway Dashboard
2. Бот должен начать работать автоматически
3. Проверьте канал - должны появиться посты

## Важные замечания:

- Railway автоматически перезапускает бота при сбоях
- Логи можно просматривать в реальном времени в Railway Dashboard
- Переменные окружения можно изменять без передеплоя
- Для парсинга новостей через Telethon может потребоваться авторизация при первом запуске (проверьте логи)

## Мониторинг:

- Следите за логами в Railway Dashboard
- Проверяйте, что бот публикует посты по расписанию
- При необходимости можно перезапустить сервис вручную
